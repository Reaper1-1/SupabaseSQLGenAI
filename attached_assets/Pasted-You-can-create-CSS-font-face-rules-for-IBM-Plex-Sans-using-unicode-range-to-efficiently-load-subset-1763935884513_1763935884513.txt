You can create CSS @font-face rules for IBM Plex Sans using unicode-range to efficiently load subsets of characters for different languages or symbols. This technique helps reduce bandwidth and optimizes font delivery, especially when serving multiple language scripts.[1][2][3]

Below are example @font-face blocks with woff2 for Latin, Cyrillic, and Greek unicode ranges:

```css
@font-face {
  font-family: 'IBM Plex Sans';
  font-style: normal;
  font-weight: 400;
  src: url('/fonts/IBMPlexSans-Regular-Latin.woff2') format('woff2');
  unicode-range: U+0000-00FF, U+0131, U+0152-0153, U+02BB-02BC, U+02C6, U+02DA, U+02DC, U+2000-206F, U+2074, U+20AC, U+2212, U+2215;
}

@font-face {
  font-family: 'IBM Plex Sans';
  font-style: normal;
  font-weight: 400;
  src: url('/fonts/IBMPlexSans-Regular-Cyrillic.woff2') format('woff2');
  unicode-range: U+0400-045F, U+0490-0491, U+04B0-04B1, U+2116;
}

@font-face {
  font-family: 'IBM Plex Sans';
  font-style: normal;
  font-weight: 400;
  src: url('/fonts/IBMPlexSans-Regular-Greek.woff2') format('woff2');
  unicode-range: U+0370-03FF;
}
```

You can get these subsetted woff2 files and recommended unicode ranges from webfont helpers or the Fontsource project. Adjust unicode ranges and file paths for your needs, repeating rules for other weights and styles as required. This setup ensures browsers only fetch the necessary glyphs for each user, boosting performance.[2][4][5][3][1]
